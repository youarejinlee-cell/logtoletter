## 1. 현재 출시 기준

- 앱 이름: `Log Planet`
- iOS Bundle ID: `com.youarejinlee.logplanet`
- Android Package: `com.youarejinlee.logplanet`
- 앱 버전: `0.1.0`
- iOS build number: `1` (`production` 빌드에서 자동 증가)
- Android version code: `1`
- 초기 타깃: iPhone 전용
- 로그인: Apple 네이티브 인증 및 카카오·Google OAuth via Supabase
- 데이터 저장: 비로그인 시 기기 로컬 저장, 로그인 시 Supabase 동기화
- 주요 탭: 기록, 모아보기, 행성, 분석 보기, 알림
- 행성 생성: 월별 기록의 카테고리, 개수, 감정 비율을 에셋 구성에 반영
- 이미지 저장: 월별 행성 이미지를 사진 보관함에 저장
- 알림: 기기 로컬 알림
- 광고, 결제, 구독: 없음

- [x] `production` 빌드 프로필이 `APP_VARIANT=production`을 사용한다.
- [x] 운영 앱 이름이 `Log Planet`으로 설정되어 있다.

- [x] 개발 빌드에서 Google 로그인과 기록 동기화를 확인했다.
- [ ] 실제 iPhone 운영 빌드에서 Apple 로그인과 기록 동기화를 확인한다.
- [ ] Preview 빌드에서 Google 로그인을 확인한다.
- [ ] Production 또는 TestFlight 빌드에서 Google 로그인을 확인한다.
- [ ] Production 또는 TestFlight 빌드에서 카카오 로그인을 확인한다.

- [ ] 계정 삭제 후 Supabase의 사용자 기록, 설정, 인증 계정이 모두 삭제되는지 확인한다.

## 5. 앱 동작 점검

### 첫 실행과 비로그인

- [x] 설치 후 첫 실행 시 이용 가이드가 표시된다.
- [x] 가이드를 닫은 뒤 7초 후 비로그인 안내가 표시된다.
- [x] 비로그인 상태에서도 기록을 기기에 저장할 수 있다.
- [x] 비로그인 기록은 로그인 전까지 행성에 반영되지 않는다.
- [x] 로그인 시 게스트 기록을 가져올지 선택할 수 있다.
- [ ] 게스트 기록 가져오기와 가져오지 않기를 실제 기기에서 각각 확인한다.
- [ ] 앱 삭제 시 비로그인 기록이 사라진다는 안내가 충분한지 확인한다.

### 기록과 캘린더

- [x] 기록 카테고리를 일, 자아성찰, 돈, 취향/취미, 관계, 건강, 기타로 분류한다.
- [x] 로그인 사용자의 기록을 Supabase에 동기화한다.
- [ ] 기록 생성, 수정, 삭제가 캘린더와 분석에 즉시 반영되는지 확인한다.
- [ ] 네트워크가 끊긴 상태에서 기록한 뒤 재연결했을 때 동기화되는지 확인한다.
- [ ] 같은 기록이 중복 생성되지 않는지 확인한다.

### 행성

- [x] `우주` 탭의 사용자 노출 이름을 `행성`으로 변경했다.
- [x] 기록 수가 0개인 카테고리는 대륙을 표시하지 않는다.
- [x] 월별 기록 수와 카테고리 순위에 따라 대륙과 레벨을 구성한다.
- [x] 기록 수가 같으면 해당 카테고리의 마지막 기록 날짜가 빠른 순으로 배치한다.
- [x] 카테고리 상세에서 레벨, 감정 비율, 전체 기록 목록을 표시한다.
- [x] 월, 기록 개수, 최다 카테고리, 행성을 이미지로 저장한다.
- [ ] 다양한 화면 크기에서 행성 배경과 에셋이 잘리지 않는지 실제 기기로 확인한다.
- [ ] 기록이 없는 달, 적은 달, 포화 상태인 달의 중심점과 레이어 순서를 확인한다.
- [ ] 사진 권한 허용, 거부, 설정 변경 후 재시도를 각각 확인한다.

### 분석 보기와 알림

- [x] 분석 보기에서 카테고리 개수·비율과 감정 비율을 볼 수 있다.
- [x] 월별 메모를 저장하고 수정할 수 있다.
- [x] 기록 알림을 설정하고 로컬 알림을 예약할 수 있다.
- [ ] 분석 슬라이드, 페이지 표시, 월 이동을 실제 데이터로 확인한다.
- [ ] 알림 권한 허용과 거부 상태를 각각 확인한다.
- [ ] 앱 재실행 후 알림 설정과 분석 메모가 유지되는지 확인한다.

## 8. 스크린샷 구성

- [ ] 모아보기: 캘린더 보기와 필터 보기가 함께 보이는 화면
- [ ] 행성: 기록이 충분히 쌓인 완성형 행성 화면
- [ ] 행성 상세: 카테고리 클로즈업과 기록 목록 화면
- [ ] 분석 보기: 카테고리·감정 분석 화면
- [ ] 알림 또는 첫 실행 가이드 화면

- [ ] 운영 빌드에서 촬영한다.
- [ ] 개인정보, 이메일, 실제 기록 내용이 노출되지 않게 테스트 데이터를 사용한다.
- [ ] 상태바 시간과 네트워크 표시를 일관되게 맞춘다.

## 10. 권장 실행 순서

1. `QA_CHECKLIST.md`를 현재 탭과 기능에 맞게 갱신한다.
