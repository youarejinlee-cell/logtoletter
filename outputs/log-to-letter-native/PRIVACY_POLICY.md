# Log Planet 개인정보 처리방침 초안

시행일: 2026년 6월 7일

`Log Planet`은 사용자가 순간의 생각과 감정을 기록하고, 지난 기록을 편지처럼 돌아볼 수 있도록 돕는 앱입니다. 이 문서는 앱이 어떤 정보를 처리하는지, 어떤 목적으로 사용하는지, 사용자가 자신의 데이터를 어떻게 관리할 수 있는지 설명합니다.

이 문서는 출시 전 검토용 초안입니다. 실제 배포 전에는 운영 주체명, 연락처, 개인정보 처리위탁 및 국외 이전 여부를 최종 확인해야 합니다.

## 1. 처리하는 개인정보

앱은 사용 방식에 따라 아래 정보를 처리할 수 있습니다.

### 사용자가 직접 입력하는 정보

- 생각 기록 내용
- 선택한 감정
- 선택한 에너지 수준
- 편지에 남긴 추신
- 알림 설정값
- 앱 설정값
  - 컬러 테마
  - 에너지 팔레트
  - 캘린더 에너지 표시 방식
  - 편지지 디자인

### 로그인 시 처리되는 정보

Google 계정으로 로그인하는 경우 아래 정보가 처리될 수 있습니다.

- 이메일 주소
- 이름 또는 표시 이름
- 프로필 이미지 URL
- Supabase 인증 사용자 ID

### 기기 권한과 관련된 정보

앱은 아래 기능을 제공하기 위해 사용자의 권한을 요청할 수 있습니다.

- 알림 권한: 기록 알림을 보내기 위해 사용합니다.
- 사진 보관함 추가 권한: 편지를 이미지로 저장하기 위해 사용합니다.
- 마이크 권한: 음성으로 기록을 남기는 기능을 사용할 때 음성을 입력받기 위해 사용합니다.
- 음성 인식 권한: 말한 내용을 기록 문장으로 변환하기 위해 사용합니다.

앱은 사용자가 권한을 허용하지 않아도 가능한 범위 내에서 기본 기능을 제공합니다.

## 2. 개인정보 처리 목적

앱은 개인정보를 아래 목적으로 사용합니다.

- 사용자가 남긴 기록을 저장하고 다시 보여주기 위해
- 캘린더에서 날짜별 기록, 감정, 에너지 흐름을 보여주기 위해
- 지난 기록을 바탕으로 편지 형태의 회고를 제공하기 위해
- 사용자가 설정한 알림 방식에 따라 기록 알림을 보내기 위해
- 로그인한 사용자의 기록과 설정을 계정에 동기화하기 위해
- 편지 이미지를 사용자의 사진 보관함에 저장하기 위해
- 앱 오류, 동기화 실패 등 사용상 문제를 확인하기 위해

## 3. 저장 방식

앱은 로그인 여부에 따라 데이터를 다르게 저장합니다.

### 비로그인 상태

비로그인 상태에서 작성한 기록과 설정은 사용자의 기기 안에 저장됩니다. 이 경우 서버 계정과 연결되지 않으며, 앱 삭제 또는 기기 저장소 초기화 시 데이터가 사라질 수 있습니다.

### 로그인 상태

사용자가 Google 계정으로 로그인하면 기록, 편지, 알림 설정, 앱 설정 일부가 Supabase 서버에 동기화될 수 있습니다. 이 기능은 여러 기기에서 데이터를 이어서 사용하거나 앱을 다시 설치했을 때 데이터를 불러오기 위해 사용됩니다.

## 4. 제3자 서비스

앱은 아래 서비스를 사용할 수 있습니다.

### Google

Google 계정 로그인을 위해 사용합니다. Google 로그인 과정에서 Google이 처리하는 정보는 Google의 개인정보 처리방침을 따릅니다.

### Supabase

사용자 인증, 데이터 저장, 서버 동기화를 위해 사용합니다. 로그인한 사용자의 기록, 편지, 설정 데이터가 Supabase에 저장될 수 있습니다.

### Apple / Expo / EAS

iOS 앱 배포, 빌드, 앱 실행 환경 제공을 위해 사용될 수 있습니다. 이 과정에서 앱 자체의 배포 및 기술 운영에 필요한 정보가 처리될 수 있습니다.

## 5. AI 기능에 대한 안내

현재 MVP에서는 편지 생성이 앱 내부 규칙 기반으로 동작합니다. 향후 AI API를 연결하여 기록을 요약하고 편지를 생성하는 기능을 제공할 수 있습니다.

AI 기능이 도입되는 경우, 아래 사항을 별도로 고지하고 개인정보 처리방침을 업데이트합니다.

- 어떤 기록 데이터가 AI 처리에 사용되는지
- 어떤 AI 제공업체를 사용하는지
- AI 제공업체가 데이터를 저장하거나 학습에 사용하는지 여부
- 사용자가 AI 기능 사용을 거부하거나 제한할 수 있는지

## 7. 보관 기간

앱은 사용자가 서비스를 이용하는 동안 기록, 편지, 설정 정보를 보관합니다.

사용자가 데이터를 삭제하거나 계정을 삭제하는 경우, 앱은 관련 데이터를 삭제합니다. 다만 법령상 보관 의무가 있거나 서비스 운영상 필요한 최소한의 기록은 해당 기간 동안 보관될 수 있습니다.

현재 앱의 기본 방침은 아래와 같습니다.

- 기기 내 데이터: 사용자가 앱에서 삭제하거나 앱을 삭제할 때까지 보관
- 서버 동기화 데이터: 사용자가 데이터 삭제 또는 계정 삭제를 요청할 때까지 보관
- 알림 설정: 사용자가 변경하거나 삭제할 때까지 보관

## 9. 보안

앱은 사용자의 기록과 계정 정보를 보호하기 위해 필요한 기술적 조치를 적용합니다.

- 로그인 사용자의 서버 데이터는 사용자별 접근 권한을 기준으로 분리합니다.
- Supabase Row Level Security 정책을 사용하여 다른 사용자의 데이터에 접근하지 못하도록 제한합니다.
- 민감한 API 키나 서버 비밀값은 앱 클라이언트에 포함하지 않는 것을 원칙으로 합니다.

다만 인터넷을 통한 데이터 전송과 저장에는 항상 일정한 위험이 존재할 수 있습니다. 사용자는 매우 민감한 개인정보, 타인의 개인정보, 금융정보, 의료정보 등을 기록할 때 주의해야 합니다.
