# AI Letter Setup

Log Planet의 주간 편지는 Supabase Edge Function `generate-letter`에서 생성한다.
앱에는 OpenAI API 키를 넣지 않는다.

## 구조

1. 앱이 기록을 `entries` 테이블에 저장한다.
2. 앱 동기화 후 `generate-letter` Edge Function을 호출한다.
3. Edge Function이 로그인한 사용자 JWT를 확인한다.
4. 아직 생성되지 않았거나 기존 규칙 기반 편지인 주간 편지만 OpenAI API로 생성한다.
5. 생성된 편지를 `letters` 테이블에 저장한다.
6. 앱이 `letters`를 다시 불러와 편지보관함에 보여준다.

이미 `prompt_version = ai-weekly-v7`인 편지는 다시 생성하지 않는다.

## Supabase Secrets

Supabase 프로젝트에서 아래 secret을 설정한다.

```bash
supabase secrets set OPENAI_API_KEY=sk-...
```

선택 사항:

```bash
supabase secrets set OPENAI_MODEL=gpt-4.1-mini
```

`OPENAI_MODEL`을 설정하지 않으면 기본값은 `gpt-4.1-mini`다.

## Deploy

프로젝트 루트에서:

```bash
cd outputs/log-planet-native
supabase functions deploy generate-letter
```

로컬 테스트:

```bash
supabase functions serve generate-letter --env-file ./supabase/.env.local
```

## App Behavior

앱은 로그인 후 전체 동기화를 할 때 `generate-letter`를 호출한다.
AI 편지 생성에 실패해도 기록 저장은 유지되고, 기존 규칙 기반 편지가 fallback처럼 남는다.

토큰 사용을 줄이기 위해 주간 기록이 아래 조건 중 하나를 만족할 때만 OpenAI를 호출한다.

- 기록이 5개 이상이다.
- 기록 본문 합계가 100자 이상이다.

두 조건을 모두 만족하지 못하면 OpenAI를 호출하지 않고 규칙 기반 편지를 저장한다.
이 경우에도 편지보관함의 요약 영역은 앱이 가진 기록 통계로 채운다.

## Prompt Policy

편지는 다음 원칙을 따른다.

- AI는 사용자의 일주일 기록을 읽고 짧은 편지를 보내는 `지적이면서도 감정을 배려해줄 줄 아는 적당한 거리감의 언니/누나`로 동작한다.
- 편지의 목적은 위로나 분석이 아니라 사용자가 자신의 일주일을 한 발 떨어져서 객관적으로 다시 보게 만드는 것이다.
- 기록을 요약하지 않고, 기록들 사이의 연결점을 관찰한다.
- 기록 전체에서 가장 눈에 띄는 흐름 1~2개만 다룬다.
- 감정과 에너지의 비율, 생각 내용과 선택한 감정/에너지 사이의 차이, 사건보다 반응과 태도를 중심으로 본다.
- 에너지는 사용자가 `사용한 에너지(%)`를 0~100%로 고른 값이다.
- 업무, 인간관계, 연인과의 관계, 가족관계, 꿈과 이상, 취향 등 자주 등장하는 분류를 알아차릴 수 있게 한다.
- 가능하면 에너지가 좋았던 때나 긍정 감정이 있었던 때를 간단히 언급하고, 그런 일의 빈도를 높여볼 수 있는지 가볍게 제안한다.
- 오전, 오후, 저녁 시간대별로 자주 느끼는 감정이나 습관이 보이면 알고 있었는지 묻는다. 기록 수가 충분하고 실제 차이가 보일 때만 쓴다.
- 부정적인 기록이나 낮은 점수는 누구에게나 있을 수 있는 것으로 다루고, 개선하고 싶은 부분이 있다면 돌아오는 주의 초점으로 삼게 한다.
- 가벼운 응원 문구는 긍정 감정이 전체 기록의 40% 이상일 때만 한 문장 이내로 허용한다.
- 본문에는 `패턴`, `키워드` 같은 분석 용어를 쓰지 않는다.
- 상담식 말투, 심리 진단, 과한 위로, 과한 공감, 시적인 표현 남용, 문학적 표현 남용, 감정의 의인화, 추상적인 문장을 피한다.
- 사용자가 남긴 기록에서 감성을 30% 정도 덜어낸 수준으로 답한다.
- `숨 고르기`, `마음의 결`, `고요히`, `묘하게`, `바람처럼`, `이면`, `흐름 속에서`, `~인지도 모르겠어`, `~같은 느낌이야`, `문득 그런 생각이 들었어`, `스며든다`, `진실`, `흥미롭다`, `흥미로운`을 쓰지 않는다.
- 편지 본문은 시작, 관찰, 낯설게 보기, 짧은 질문이나 생각거리의 흐름을 자연스럽게 따른다.
- 본문은 제목, 소제목, 분석 항목, 불릿포인트, 날짜 목록 없이 편지 본문만 작성한다.
- 편지 본문은 한국어 400~700자 정도로 작성한다.
- 기록이 5개 미만이고 본문 합계가 100자 미만인 경우 OpenAI를 호출하지 않고, 이전 편지를 돌아보게 하는 fallback 편지를 사용한다.
- `title`, `keyword`, `themes`, `recommendations`는 앱 UI 표시용 메타데이터로만 사용하고 본문 안에 반복하지 않는다.
