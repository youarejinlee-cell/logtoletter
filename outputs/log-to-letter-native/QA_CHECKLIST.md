# Log to Letter QA Checklist

출시 전 QA는 아래 순서대로 실제 앱에서 진행한다.

상태 표기:

- `[ ]` 미확인
- `[x]` 통과
- `[!]` 이슈 있음
- `[-]` 이번 빌드에서 해당 없음

테스트 기준:

- iOS 시뮬레이터 1회
- 실제 기기 또는 preview 빌드 1회
- 로그인 상태 1회
- 비로그인 상태 1회

## 0. 빌드 전 점검

| 상태 | 테스트 | 절차 | 기대 결과 | 메모 |
| --- | --- | --- | --- | --- |
| [x] | TypeScript 검사 | `PATH=/opt/homebrew/opt/node@22/bin:$PATH npx tsc --noEmit` 실행 | 오류 없이 통과 | 2026-06-07 통과 |
| [ ] | 앱 실행 | `npx expo start -c` 후 시뮬레이터에서 실행 | 앱이 splash 이후 정상 진입 |  |
| [ ] | 상단 안전영역 | 앱 첫 화면 확인 | 상단 타이틀이 상태바/다이내믹 아일랜드와 겹치지 않음 |  |
| [ ] | 하단 탭 | 모든 탭 이동 | 하단 탭이 화면 하단 UI와 겹치지 않음 |  |
| [ ] | 기본 스크롤 | 기록, 캘린더, 편지보관함, 알림, 설정 화면 스크롤 | 긴 화면에서 스크롤이 막히지 않음 |  |
